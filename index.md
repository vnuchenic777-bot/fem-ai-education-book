# Проектирование цифровой образовательной экосистемы факультета

**Методология, архитектура и реализация курса ДПО по ИИ-инструментам**

Короткое название книги: **«Цифровая образовательная экосистема ФЭМ»**.

Эта Jupyter Book-книга собирает исследовательско-методические, инженерные и структурированные материалы проекта по созданию цифровой образовательной платформы факультета и курса ДПО «ИИ-инструменты в преподавательской, научной и повседневной работе: от понимания принципов до практического применения».

## Научно-методический замысел

Проект рассматривает образовательную платформу не только как набор экранов и backend-сущностей, а как воспроизводимую экосистему: курс, компетенции, учебный план, карта уроков, фонд оценочных средств, итоговый проект, платформенная интеграция, данные, роли, статусы и документ об обучении должны быть согласованы между собой.

Формат Jupyter Book выбран для того, чтобы сохранить проверяемость материалов: Markdown-главы остаются читаемыми, Jupyter Notebook-главы могут исполняться, а YAML/JSON-файлы сохраняют вложенные таблицы, матрицы, рубрики, модели и данные без превращения в обычный текст.

## Структура книги

- **Введение** — обзор проекта, контекст исследования и словарь.
- **Архитектура платформы** — продуктовая логика, платформенная интеграция, модель данных и API.
- **Курс ДПО по ИИ-инструментам** — концепция курса, компетенции ПК-ИИ-1 — ПК-ИИ-10, учебный план, карта уроков, модули, итоговый проект и фонд оценочных средств.
- **Интеграция ИИ в образовательную деятельность** — стратегия факультета, задания с обязательным использованием ИИ, безопасность, этика и обновление РПМ.
- **Разработка платформы** — backend, PWA, роли, доступ и хранение.
- **Сертификат или удостоверение** — модель документа об обучении, данные, жизненный цикл и QR-проверка.
- **Научно-публикационный контур** — черновики будущих статей.
- **Приложения** — решения, технические спецификации, открытые вопросы и библиография.

## Важное предупреждение

Материалы книги являются исследовательско-методическими и техническими. Они не утверждают юридический статус документов об обучении, порядок обработки персональных данных, локальные правила образовательной организации или нормативные основания выдачи сертификатов и удостоверений. Все юридически значимые положения требуют отдельной проверки и согласования с образовательной организацией.

Раздел о документе об обучении находится здесь: [](book/05_certification/certificate_model.md).

## Как читать эту книгу

Сначала полезно прочитать обзор проекта и контекст исследования. Затем можно перейти к курсу ДПО: концепции, компетенциям, учебному плану и фонду оценочных средств. Инженерную часть лучше читать после методической, потому что модели данных, роли и статусы опираются на образовательную структуру. Раздел о сертификате или удостоверении лучше рассматривать как техническую и методическую модель, а не как утвержденный юридический регламент.

## Как воспроизвести книгу локально

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python scripts/validate_data.py
python scripts/check_book.py
jupyter-book build .
python -m http.server --directory _build/html 8000
```

Для Windows:

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python scripts\validate_data.py
python scripts\check_book.py
jupyter-book build .
python -m http.server --directory _build/html 8000
```
