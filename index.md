# Проектирование цифровой образовательной экосистемы факультета

**Методология, архитектура и реализация курса ДПО по ИИ-инструментам**

Короткое название книги: **«Цифровая образовательная экосистема ФЭМ»**.

Автор: **Морозов Артём Михайлович**, младший научный сотрудник, преподаватель курса «Введение в ИТ», кафедра «Бизнес-информатики», СПБГТИ(ТУ).

Эта Jupyter Book-книга представляет исследовательско-методический и инженерный контур проекта по созданию цифровой образовательной платформы факультета и курса ДПО «ИИ-инструменты в преподавательской, научной и повседневной работе: от понимания принципов до практического применения».

## Научно-методический замысел

Проект рассматривает образовательную платформу не как набор разрозненных экранов, а как воспроизводимую экосистему. В этой экосистеме курс, компетенции, учебный план, карта уроков, фонд оценочных средств, итоговый проект, данные, роли, статусы, аудит и документ об обучении должны быть согласованы между собой.

Публикационная цель книги состоит в том, чтобы показать путь разработки и внедрения платформы в форме открытого академического отчета: с описанием методологии, архитектурной логики, образовательных моделей, ограничений, проверяемых данных и визуализаций, но без раскрытия ключей доступа, персональных данных и внутренних материалов, не предназначенных для публичного размещения.

## Структура книги

- **Введение** — обзор проекта, контекст исследования и словарь.
- **Архитектура платформы** — продуктовая логика, платформенная интеграция, доменная модель данных и обзор взаимодействия компонентов.
- **Курс ДПО по ИИ-инструментам** — концепция курса, компетенции ПК-ИИ-1 — ПК-ИИ-10, учебный план, карта уроков, модули, итоговый проект и фонд оценочных средств.
- **Интеграция ИИ в образовательную деятельность** — стратегия факультета, задания с обязательным использованием ИИ, безопасность, этика и обновление РПМ.
- **Разработка платформы** — публично допустимое описание пользовательского контура, ролей, доступа, хранения и технических границ.
- **Сертификат или удостоверение** — модель документа об обучении, юридический статус, данные, жизненный цикл и QR-проверка.
- **Научно-публикационный контур** — публикационные версии статей и материалы для последующей журнальной редакции.
- **Приложения** — решения, технические спецификации, границы открытой публикации, открытые вопросы и библиография.

## Важное предупреждение

Материалы книги являются исследовательско-методическими и техническими. Они не утверждают юридический статус документов об обучении, порядок обработки персональных данных, локальные правила образовательной организации или нормативные основания выдачи сертификатов и удостоверений. Все юридически значимые положения требуют отдельной проверки и согласования с образовательной организацией.

Раздел о документе об обучении находится здесь: [](book/05_certification/certificate_model.md). Отдельная проектная позиция по юридическому статусу вынесена в раздел [](book/05_certification/legal_status.md).

## Как читать эту книгу

Сначала полезно прочитать обзор проекта и исследовательский контекст. Затем можно перейти к курсу ДПО: концепции, компетенциям, учебному плану и фонду оценочных средств. Инженерную часть лучше читать после методической, потому что модели данных, роли и статусы опираются на образовательную структуру. Раздел о сертификате или удостоверении следует рассматривать как техническую и методическую модель, а не как утвержденный юридический регламент.

## Как воспроизвести книгу локально

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python scripts/validate_data.py
python scripts/check_book.py
jupyter-book build .
python -m http.server --directory _build/html 8000
```

Для Windows:

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python scripts\validate_data.py
python scripts\check_book.py
jupyter-book build .
python -m http.server --directory _build/html 8000
```
