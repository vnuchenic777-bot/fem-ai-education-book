# Импорт пакета 08_FINAL_PROJECT_DPO_AI

Рекомендуемый порядок использования:

1. Для Claude / Codex / ChatGPT / IDE открыть файл:
   - `08_FINAL_PROJECT_DPO_AI_IMPLEMENTATION_SPEC.md`

2. Для машинной обработки, генерации задач, импорта в проект или проверки структуры использовать:
   - `08_FINAL_PROJECT_DPO_AI_IMPLEMENTATION_SPEC.yaml`
   - `08_FINAL_PROJECT_DPO_AI_IMPLEMENTATION_SPEC.json`

3. Почему не DOCX:
   - DOCX плохо подходит для инженерной среды;
   - таблицы могут быть потеряны или неверно распознаны;
   - YAML/JSON сохраняют вложенные структуры, рубрики, критерии и требования без потери колонок.

4. Каноническим источником структуры считать YAML/JSON.
   Markdown является удобной читаемой версией того же ТЗ.
