# API-модель

## Назначение

Раздел должен описывать публично допустимую модель API образовательной платформы: какие операции нужны для курса, оценивания, прогресса, сертификата или удостоверения и QR-проверки.

## Статус

Черновик.

## Что нужно перенести из веток

- Из ветки 11: задачи платформенной интеграции и карты загрузки контента.
- Из ветки 10: `certificate_api_contracts.json` и модель публичной проверки документа.
- Из ветки 5: операции по оценочным элементам, попыткам, проверкам и статусам.

## Ограничение

API-модель в этой книге не должна раскрывать закрытые ключи, внутренние адреса, персональные данные или сведения, не предназначенные для публичной публикации.
